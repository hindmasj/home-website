Getting Started With Bootstrap

A first effort at using Bootstrap to manage the page layout with less effort and more style. Introduces how to download and install the code then create a simple layout for the page.

Enabling Your Site

    Steps to getting your web pages bootstrapped.
  1. Download the zip file from the getting started page of the bootstrap site.
  2. Go to /var/www/html and unzip the file.
  3. (SELinux Only) Assigned the correct label "httpd_sys_content_t" to the new directory. 
    chcon -R -t httpd_sys_content_t /var/www/html/bootstrap-*-dist
  4. Create a soft link "bootstrap" in the same directory to the unzipped directory.
  5. Add this text to the head section. Make sure these declarations come before any of your own declarations. 
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
<link rel="stylesheet" type="text/css" href="/bootstrap/css/bootstrap.min.css">
  6. Add this text to the bottom of the page 
    <script src="http://code.jquery.com/jquery.min.js"></script>
<script src="/bootstrap/js/bootstrap.min.js"></script>

Creating A Layout

    This page has a simple layout with a header (the title and some introductory text), some rows and columns (each section) and a footer (the navigation section).

  1. Sorround all of the page that you want to be bootstrapped with a "div" tag with a class of "container". 
    <div class="container">
</div>
  2. With the "container", the header can be picked out by sourrounding it with a div of the "jumbotron" class. 
    <div class="jumbotron">
<h1>Big Heading</h1>
<p>My heading text</p>
</div>
  3. The sections that have equal prominence should be wrapped in a "row" div, and then each section is wrapped in its own "col-xs-*" div. The "*" represents how wide the column should be, between 1 and 12. The maximum number of columns in a row is 12, so by setting these values you govern how these columns rearrange themselves as the width of the browser changes. 
    <div class="row">
<div class="col-xs-4">
<p>Section 1 text.</p>
</div>
<div class="col-xs-4">
<p>Section 2 text.</p>
</div>
<div class="col-xs-4">
<p>Section 3 text.</p>
</div>
</div>
  4. The navigation section uses a "nav" and a "navbar" tag. See the next section for more details.

Creating A Navbar

    Add a navbar to the page to replace the simple links.
  1. Use the HTML5 "nav" tag to denote the area for navigation controls. The bootstrap classes "navbar" and "navber-default" create a default type of navbar. 
    <nav role="navigation" class="navbar navbar-default">
</nav>
  2. Create a navbar header. This serves two purposes: allows the navbar to collapse when shown on a tiny screen; and adds a branding section where you put a button that explains the role of the navbar. The collapse button is only visible on tiny screens. Note the use of the "sr-only" span which contains a text label that should only be vissible to screen readers; and the 3 "icon-bar" spans which create the 3 short lines that create the now common menu collapsed icon. The target of the collapse button is the actual menu buttons. 
    <div class="navbar-header">
<button type="button" data-target="#navbarCollapse" data-toggle="collapse" class="navbar-toggle">
<span class="sr-only">Toggle navigation</span>
<span class="icon-bar"></span>
<span class="icon-bar"></span>
<span class="icon-bar"></span>
</button>
<a href="#navbarCollapse" class="navbar-brand">Navigation</a>
</div>
  3. Add the actual navbar buttons in an unordered list. The list is sorrounded by a div which declares the list to be collapsable and provides a target for the menu button. 
    <div id="navbarCollapse" class="collapse navbar-collapse">
<ul class="nav navbar-nav">
<li><a href="..">Index</a></li>
<li><a href="/">Site</a></li>
<li><a href="http://home.btconnect.com/sjhindmarch/">External Home Page</a>
</li>
</ul>
</div>

Improving The Layout

    The layout where everything is in one row looks a bit cramped and there is now no margin around the text where they line up against each other. Each section is a different size and the effect makes the text look chequered.
  1. A better layout is to give each section its own row and then use empty columns to pad out a margin. At the same time add some comments to the closing "div" tags so that you can keep track of them. 
    <div class="row">
<div class="col-xs-1"></div>
<div class="col-xs-10">
<h2>Improving The Layout</h2>
</div> <!-- col improving the layout-->
<div class="col-xs-1"></div>
</div> <!-- row -->
  2. Now each section is spaced out so that it stands out on its own. The source is easier to read and you have less risk of putting one div inside the wrong one in the hierarchy.
  3. The Jumbotron looks a little cramped at the top so add an empty row to separate it from the top of the screen. Just use a paragraph tag to force the row to occupy some space. 
    <div class="row"><p></p></div>
  4. To maintain the indented look of each paragraph, and to emphasise the instructional look of the site, each section is one ordered list. The main text goes in the header section and every paragraph after is a list item. 
    <ol>This is the main text
<li>This is next para ...</li>
<li>... and this is some more</li>
</ol>

Fixing The Style

    The style of this site is trying to look like an antique document - sort of. The background colour is "burlywood" which looks a little like aged paper. The headings are "darkred" and have some "red" shadow to look a little like illuminated text. The Jumbotron and Navbar however have not picked up on this style. They both have a grey background and the Jumbotron text is black instead of read, even though it has picked up the shadow effect.
  1. To fix the Jumbotron set a style for the class "jumbotron" with a new background colour. I have chosen "lightgoldenrodyellow" for now as I like the lighter contrast with the background. 
    .jumbotron {
    background-color:lightgoldenrodyellow;
}
  2. To fix the Jumbotron heading colour a separate style is needed for the "h1" element. 
    .jumbotron h1{
    color:darkred;
}
  3. And the navbar background colour is set the same way for the "navbar" class. 
    .navbar {
    background-color:lightgoldenrodyellow;
}
