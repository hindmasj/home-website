Enabling Your Site

    Steps to getting your web pages bootstrapped.
  1. Download the zip file from the getting started page of the bootstrap site.
  2. Go to /var/www/html and unzip the file.
  3. (SELinux Only) Assigned the correct label "httpd_sys_content_t" to the new directory. 
    chcon -R -t httpd_sys_content_t /var/www/html/bootstrap-*-dist
  4. Create a soft link "bootstrap" in the same directory to the unzipped directory.
  5. Add this text to the head section. Make sure these declarations come before any of your own declarations. 
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
<link rel="stylesheet" type="text/css" href="/bootstrap/css/bootstrap.min.css">
  6. Add this text to the bottom of the page 
    <script src="http://code.jquery.com/jquery.min.js"></script>
<script src="/bootstrap/js/bootstrap.min.js"></script>

Creating A Layout

    This page has a simple layout with a header (the title and references), some columns (each section) and a footer (the navigation section).

  1. Sorround all of the page that you want to be bootstrapped with a "div" tag with a class of "container". 
    <div class="container">
</div>
  2. With the "container", the header can be picked out by sourrounding it with a div of the "jumbotron" class. 
    <div class="jumbotron">
<h1>Big Heading</h1>
<p>My heading text</p>
</div>
  3. The sections that have equal prominence should be wrapped in a "row" div, and then each section is wrapped in its own "col-xs-*" div. The "*" represents how wide the column should be, between 1 and 12. The maximum number of columns in a row is 12, so by setting these values you govern how these columns rearrange themselves as the width of the browser changes. 
    <div class="row">
<div class="col-xs-4">
<p>Section 1 text.</p>
</div>
<div class="col-xs-4">
<p>Section 2 text.</p>
</div>
<div class="col-xs-4">
<p>Section 3 text.</p>
</div>
</div>
  4. The navigation section uses a "nav" and a "navbar" tag. See the next section for more details.

Creating A Navbar

    Add a navbar to the page to replace the simple links.
  1. Use the HTML5 "nav" tag to denote the area for navigation controls. The bootstrap classes "navbar" and "navber-default" create a default type of navbar. 
    <nav role="navigation" class="navbar navbar-default">
</nav>
  2. Create a navbar header. This serves two purposes: allows the navbar to collapse when shown on a tiny screen; and adds a branding section where you put a button that explains the role of the navbar. The collapse button is only visible on tiny screens. Note the use of the "sr-only" span which contains a text label that should only be vissible to screen readers; and the 3 "icon-bar" spans which create the 3 short lines that create the now common menu collapsed icon. The target of the collapse button is the actual menu buttons. 
    <div class="navbar-header">
<button type="button" data-target="#navbarCollapse" data-toggle="collapse" class="navbar-toggle">
<span class="sr-only">Toggle navigation</span>
<span class="icon-bar"></span>
<span class="icon-bar"></span>
<span class="icon-bar"></span>
</button>
<a href="#navbarCollapse" class="navbar-brand">Navigation</a>
</div>
  3. Add the actual navbar buttons in an unordered list. The list is sorrounded by a div which declares the list to be collapsable and provides a target for the menu button. 
    <div id="navbarCollapse" class="collapse navbar-collapse">
<ul class="nav navbar-nav">
<li><a href="..">Index</a></li>
<li><a href="/">Site</a></li>
<li><a href="http://home.btconnect.com/sjhindmarch/">External Home Page</a>
</li>
</ul>
</div>
